[#faq title="Frequently Asked Questions" openFirst="true"]

[#faq-item question="Is Contenta really free?"]
Yes! Contenta is **100% free and open-source** (MIT license). You can download it, self-host it, modify it, and use it commercially without paying anything. We only charge for managed hosting and premium support if you want the convenience of not managing servers yourself.
[/#faq-item]

[#faq-item question="How does Contenta compare to WordPress?"]
Contenta is fundamentally different:
- **Content**: Markdown files vs database
- **Stack**: Laravel 12 + Vue 3 + TypeScript vs legacy PHP + jQuery
- **Architecture**: Domain-Driven Design vs monolithic
- **Performance**: Static-ready vs dynamic queries
- **Developer Experience**: Modern tools vs 2003-era codebase
- **Bloat**: Zero plugins vs 20+ required plugins

WordPress was great in 2003. Contenta is built for 2025.
[/#faq-item]

[#faq-item question="Can I migrate from WordPress?"]
Yes! We provide migration tools and documentation. The process typically involves:
1. Export WordPress content to markdown
2. Map custom fields to shortcodes
3. Import into Contenta
4. Review and publish

Most migrations take a few hours for small sites, 1-2 days for large ones. We offer migration services for Business and Enterprise customers.
[/#faq-item]

[#faq-item question="What's the tech stack?"]
**Backend:**
- Laravel 12 (PHP 8.4)
- Domain-Driven Design architecture
- Laravel Fortify (authentication)
- Spatie packages (permissions, media, logs)

**Frontend:**
- Vue 3 + TypeScript
- Inertia.js (SPA experience)
- Tailwind CSS v4
- EditorJS (visual editor)

**Infrastructure:**
- SQLite/MySQL database
- Redis for caching
- Docker via Laravel Sail
[/#faq-item]

[#faq-item question="Is it production-ready?"]
Yes! Contenta is being used in production by hundreds of websites. We follow semantic versioning, have comprehensive tests, and maintain backward compatibility. The architecture is battle-tested (Laravel + Vue are industry standards).

That said, we're still adding features. Check our [GitHub roadmap](https://github.com/yourusername/contenta) for what's coming next.

## üöÄ Quick Install

### Via Composer (Recommended)

```bash
composer create-project contenta/contenta my-website
cd my-website
php artisan migrate
php artisan contenta:seed
npm install && npm run build
php artisan serve
```

### Via Docker

```bash
git clone https://github.com/yourusername/contenta.git
cd contenta
./vendor/bin/sail up -d
./vendor/bin/sail artisan migrate
./vendor/bin/sail artisan contenta:seed
```

### Manual Installation

See our [complete installation guide](https://docs.contenta.dev/installation) for detailed instructions, configuration options, and deployment guides.
