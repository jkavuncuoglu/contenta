# Contributing

[← Back: Releases & Forking](RELEASES.md) • [Read next: Legal →](LEGAL.md)

- Fork the repo and create a feature branch from `main`.
- Follow existing coding styles for PHP (Laravel) and Vue/TypeScript.
- Add or update tests where applicable.
- Run linters and builds before opening a PR.
- Describe changes clearly and reference related issues.
